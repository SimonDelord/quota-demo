Kubernetes quota viewer

Reads ResourceQuota objects via the Kubernetes API (same information as kubectl describe resourcequota) and charts usage versus hard limits.

Quota used counts only admitted Pods. If scaling creates Pods that are rejected by quota, those Pods never exist, so CPU/memory counters often do not move—watch Deployments below for desired versus ready replicas.




Tip: turn on Auto-refresh during demos. When quota blocks new Pods, limits.cpu / requests.cpu may stay flat while desired replicas > ready.
